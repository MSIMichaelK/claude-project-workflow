## How to Use This Document

1. **When investigating something complex**, create a belief entry (B-xxx) for each assumption
2. **Record evidence** for and against — be specific, cite data, timestamps, or observations
3. **Design tests** (T-xxxN) to validate or disprove each belief
4. **Update status** as tests are run: ✅ CONFIRMED, 🔴 WRONG, ⚠️ PARTIAL, 🟡 OPEN
5. **Never delete entries** — if a belief is proved wrong, mark it 🔴 WRONG and explain why. This prevents re-investigating the same thing.

### Status Key

| Status | Meaning |
|--------|---------|
| ✅ CONFIRMED | Validated with evidence. Treat as fact. |
| 🔴 WRONG | Disproved. Do NOT rely on this assumption. |
| ⚠️ PARTIAL | Partially correct — see details for nuance. |
| 🟡 OPEN | Not yet tested. May or may not be true. |
